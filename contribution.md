
## git commit message type
- feat：開發新功能
- fix：除錯(如有報案來源，以#號備註在後)
- docs：文件類（例如README）
- style： 格式（例如空格、格式縮排等等，不改變代
碼邏輯）
- refactor：代碼重構（非新功能或修改bug）
- test：測試
- chore：構建過程或輔助工具的變動


## css預處理
- 巢狀不超過3層，含偽元素不超過4層
- 以scss為例，內容不多請按照順序寫在一起，編
寫順序為：
    - variable 變數
    - function 管理(計算)值
    - extend 繼承 (管理樣式)
    - mixin 管理屬性
